# UI/UX 设计方针

这仅作为备忘录，旨在分享 VOICEVOX 的 UX 和 UI 设计方针。

## 注意事项等

此文件更新遗漏的可能性较高。请同时查阅实际文件。

如果不知道各 UI 的名称，请参阅[VOICEVOX 专用 UI 名称](./UX・UIデザインの方針.md#voicevox-専用-ui-の名称)。

## 基本方针

- 我们最重视的是“用户能够完成他们想要创作的内容”。
- 基于这一理念，我们在设计时注重以下几点：
  - 让人感觉想使用
  - 操作流程清晰明了
  - 操作无需思考
- 此外，我们也重视“让用户制作出更好的内容”。
- 基于这一理念，我们也注重以下几点：
  - 轻松提高语音质量
  - 软件具有创造性

下面介绍具体是如何落实到实现中的。

- 让人感觉想使用
  - 为了让软件尽可能触达更多人，我们免费发布，并尽可能提高质量。
  - 为了避免让人觉得过时，我们追求新颖的视觉设计。
  - 为了避免给未来的用户留下负面印象，我们在社交媒体上避免负面情绪，并积极解决用户的负面疑问。
- 操作流程清晰明了
  - 我们注重首屏设计，使其能实现从文本输入到语音导出的功能。
  - 对于可有可无的功能，我们尽量将其 UI 设计得不那么显眼。
- 操作无需思考
  - 我们注重简洁的 UI。
  - 避免使用难懂的词语，注重简洁的文案。
  - 设计 UI 时，即使不知道快捷键也能使用。
- 轻松提高语音质量
  - 首屏设计能让用户了解可以调整语音。

### 使命与软件方针的对应

- 我们认为 UGC（用户生成内容）对于实现[使命](./ミッション・バリュー・ビジョン.md#ミッション)至关重要。
  - 这种理念是用户创作的内容能够扩大圈子。
- 基于这一理念，我们的软件方针是促进更多更好的内容创作。

## 设计决策流程

尽管仍在不断尝试和摸索，但大致流程如下：

1.  通过 Issue 等方式提出设计外观。
    - 这是为了尽早发现遗漏考虑的步骤。
    - 文本、手绘图、示例实现都可以。
2.  实际实现并创建 Pull Request。
    - UI/UX 设计有很多部分需要实际操作才能理解，因此需要进行实现。
    - 设计变更即使代码量不多，内容也可能很庞大，因此如果能将总差异控制在 +/- 80 行左右，合并会更顺利。
3.  找出问题点。
4.  合并。
5.  审视整体设计，如有需要，由维护者进行微调。

## 详细设计指南

- 取消按钮在左侧。
- 对话框和吐司提示的区分使用：
  - 对于由用户操作引起且错误或频率较低的反馈，显示对话框（`SHOW_DIALOG_`系列）。
    - 例：[需要重启引擎] 引擎已停止，无法生成语音。是否重启引擎？[不重启/重启]
  - 除此之外，通过吐司提示通知（`SHOW_NOTIFY_`系列）。
    - 例：已导出语音。
    - 例：引擎异常终止。需要重启引擎。
- 关于对话框：
  - 警告和确认的区分使用：
    - 如果倾向于继续操作，使用确认。
    - 如果倾向于取消操作，使用警告。
  - 提问对话框的文案：
    - 标题为疑问句“～吗？”
    - 正文简洁，避免使用“可以吗？”等。
  - 错误对话框的文案：
    - 标题为陈述句“〜失败了。”
  - 按钮颜色：
    - 肯定且选择可能性高的按钮使用主色。
    - 具有破坏性且恢复状态耗时的操作按钮使用警告色。

### 其他概念名称

- 向用户展示时，使用“文件夹”而非“目录”。
- 多引擎相关：
  - 符合 VOICEVOX API 的引擎称为“引擎”。
  - 如果不确定是 VOICEVOX 引擎还是符合标准的引擎，则称为“VOICEVOX 兼容引擎”。
  - VVPP 文件直接称作“VVPP 文件”，如果需要说明，则称作“VOICEVOX 兼容引擎包”。
- 采用与 VOICEVOX 编辑器相同 UI 的软件统称为“VOICEVOX 系列软件”。
  - 正式名称为“VOICEVOX 系统 UI 软件”，因过长几乎不用。

### 文体

- 字母使用半角。
- 括号使用全角。
  - 但如果内容全部是半角字符，括号也使用半角。
- 使用敬体（“是/有”体）。
- 项目名称以名词结尾（体言止）。
- 按钮内容为“～（动词）”。

### 工具提示 (tooltip)

- 句末加句号。
- 如果句末有括号，句号放在括号外面。
  - 例：这样做（但需要注意）。
- 单选框为“可选择。”
- 选择框为“可更改。”
- 按钮为“可～。”
- 复选框
  - XX 功能为“启用 XX 功能。将～～。”
  - 如果默认关闭，则为“如果开启，将～～/不会～～。”
  - 如果默认开启，则为“如果关闭，将不会～～/会～～。”
